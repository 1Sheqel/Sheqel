# SheqelMotion Studio — Презентация для руководства

---

## 1. Что это за продукт

**SheqelMotion Studio** — десктопное приложение (macOS / Windows / Linux) для автоматизированного производства лип-синк видео.

Ключевая задача: замена ручного монтажа. Оператор загружает голосовую дорожку и видео — приложение само синхронизирует движение губ и выдаёт финальный ролик студийного качества.

---

## 2. Функциональные возможности

| Функция | Описание |
|---|---|
| **Режим «1 видео»** | Один голос → одно видео, лип-синк целиком |
| **Режим «Начало + Конец»** | Голос автоматически режется на 2 части, каждая накладывается на своё видео |
| **Генерация голоса** | Прямая интеграция с ElevenLabs (TTS, 44 100 Гц, 192 kbps MP3) |
| **Клонирование голоса** | Загрузка аудио-сэмпла → создание клона в ElevenLabs |
| **Очередь роликов** | До N роликов параллельно, пауза между запусками настраивается |
| **Автообновление** | Приложение само проверяет GitHub и предлагает обновиться |
| **Лог в реальном времени** | Отдельное окно прогресса для каждого шага обработки |

## 3. Технологический стек

```
Python 3          — язык приложения
CustomTkinter     — современный UI (тёмная тема, адаптивное окно)
Pillow (PIL)      — генерация неон-логотипа в заголовке
Requests          — HTTP-запросы к ElevenLabs и Sync.so API
FFmpeg / FFprobe  — обработка аудио и видео
ThreadPoolExecutor — параллельная очередь роликов
ElevenLabs API    — синтез и клонирование голоса
Sync.so API       — лип-синк на нейросети (модель sync-3)
Litterbox.catbox  — временный хостинг файлов для передачи в Sync (1 ч, автоудаление)
```

## 4. Архитектура и поток данных

```
Пользователь
     │
     ▼
[Интерфейс CTk]
     │
     ├─ Кнопка «Сгенерировать голос»
     │       → ElevenLabs TTS API → full_voice.mp3 → Рабочий стол
     │
     ├─ Кнопка «НАЧАТЬ ОЧЕРЕДЬ»
     │       → ThreadPoolExecutor (2 параллельных потока)
     │               │
     │               ├─ Режим «1 видео»
     │               │   → конвертация аудио в WAV
     │               │   → подготовка видео (trim без перекодировки)
     │               │   → upload видео + аудио на litterbox (временный URL)
     │               │   → Sync.so API (polling до COMPLETED)
     │               │   → enhance_video (sharpen, x264 CRF 13, AAC 256k)
     │               │   → финальный .mp4 на Рабочий стол
     │               │
     │               └─ Режим «Начало + Конец»
     │                   → detect_silences (ffmpeg silencedetect)
     │                   → split audio → part_start.wav + part_end.wav
     │                   → 2× lipsync (параллельно внутри одного ролика)
     │                   → 2× enhance_video
     │                   → 2 финальных .mp4 на Рабочий стол
     │
     └─ Автообновление (каждые 3 сек после запуска)
             → проверка GitHub manifest → предложение обновиться
```

## 5. Качество выходного видео

| Параметр | Значение |
|---|---|
| Кодек видео | H.264 (libx264) |
| CRF (качество) | 13 (очень высокое, 0 = без потерь) |
| Preset | slow (максимальное сжатие при высоком качестве) |
| Sharpening | unsharp 3×3, 0.6 — повышение чёткости после лип-синк |
| Аудио | AAC 256 kbps, 44 100 Гц |
| Формат | yuv420p, faststart (совместим с любым плеером и соцсетями) |

## 6. Конфигурация и хранение данных

- API ключи (ElevenLabs, Sync.so) сохраняются в `~/.version.json`
- Все временные файлы удаляются после обработки (папка `.tmp`)
- Результаты сохраняются в `~/Desktop/Lipsync_Queue_Output/<batch>/`

## 8. Анализ кода — найденные проблемы

### Проблемы безопасности

| Приоритет | Описание |
|---|---|
| 🔴 ВЫСОКИЙ | Автообновление скачивает и исполняет `app.py` с GitHub без проверки hash-суммы или подписи. Компрометация репозитория = RCE на всех машинах пользователей. |
| 🟡 СРЕДНИЙ | API ключи хранятся в `~/.version.json` в открытом тексте (JSON). |
| 🟡 СРЕДНИЙ | Видео и аудио файлы загружаются на публичный хостинг litterbox.catbox.moe — пользователь должен быть явно уведомлён об этом. |
| 🟢 НИЗКИЙ | Имя конфиг-файла `~/.version.json` вводит в заблуждение (выглядит как системный файл версий). |

### Качество кода

| Тип | Описание |
|---|---|
| Дублирование | `fetch_voice_id_by_name` и `fetch_all_voices` оба делают один и тот же GET `/v1/voices`. Первый можно реализовать через второй. |
| Дублирование | `get_roll_title` и часть `render_roll_card` (строки 1398–1406) делают одно и то же. |
| Архитектура | `global SYNC_API_KEY` используется в 3 разных местах, `global ELEVENLABS_API_KEY` — в 2. Плохая практика. |
| Стиль | `import tempfile` внутри метода `play_voice_preview` (строка 1117) — должен быть вверху файла. |
| Отступы | Тело метода `queue_thread` имеет лишний уровень отступа (строка 1738). |
| Комментарии | Отладочные комментарии `# ← ДОБАВИТЬ ЭТОТ БЛОК` и `# ← КОНЕЦ БЛОКА` оставлены в продакшн коде. |

## 9. Рекомендации

1. **Добавить hash-проверку при автообновлении** — манифест должен содержать SHA256 нового файла.
2. **Обновить `requests`** до 2.34.2 (`pip install requests==2.34.2`).
3. **Переименовать конфиг** с `.version.json` на `.sheqelmotion.json`.
4. **Уведомить пользователя** о загрузке файлов на litterbox.catbox.moe в явном диалоге.
5. **Вынести `import tempfile`** наверх файла.
6. **Удалить отладочные комментарии** из кода.
